# SESSF Systems Map Connection Analysis Prompt

## Purpose
This prompt guides the systematic analysis of isolated variables in the SESSF systems map to identify evidence-based connections and relationships, ensuring a more cohesive and interconnected representation of the fishery system.

## Process Structure

### Step 1: Isolated Variable Identification
1. List all currently isolated variables in the systems map
2. For each isolated variable:
   - Document its category (Environmental, Economic, Social, Technical, Market, Indigenous)
   - Note its current definition and measurement units from Phase 2 documentation
   - Record any existing evidence about its behavior or importance from the five phases

### Step 2: Primary Connection Analysis
For each isolated variable:

1. **Direct Effects Search**
   - Review all five phases to identify explicit mentions of relationships with other variables
   - Document evidence of direct causal relationships
   - Note the direction (positive/negative/variable) and strength (strong/medium/weak) of relationships
   - Record specific citations from the documentation

2. **System Role Analysis**
   - Identify the variable's role in:
     * Environmental dynamics (Phase 1 & 2)
     * Economic processes (Phase 5)
     * Management frameworks (Phase 1)
     * Social systems (Phase 1)
     * Technical operations (Phase 2)

3. **Temporal Analysis**
   - Consider relationships across different timeframes:
     * Short-term operational connections
     * Medium-term tactical relationships
     * Long-term strategic linkages
   - Document evidence of time delays and feedback loops from Phase 3

### Step 3: Secondary Connection Analysis

1. **Pathway Mapping**
   - Identify potential intermediate variables that could connect isolated variables to the main system
   - Consider variables that act as:
     * Environmental mediators
     * Economic linkages
     * Social connectors
     * Technical bridges
     * Market interfaces

2. **Cross-domain Integration**
   - Look for connections across different variable categories
   - Consider how the variable might influence or be influenced by:
     * Climate adaptation pathways (Phase 4)
     * Market integration mechanisms (Phase 5)
     * Management interventions (Phase 4)
     * Social-ecological interactions (Phase 1)

### Step 4: Evidence Validation

For each proposed connection:

1. **Documentation Review**
   - Locate specific evidence in the five phases supporting the connection
   - Rate evidence quality (High/Medium/Low)
   - Note any uncertainties or knowledge gaps

2. **Relationship Characterization**
   - Define the nature of the relationship:
     * Direction (positive/negative/variable)
     * Strength (strong/medium/weak)
     * Confidence level based on available evidence
     * Time delays or conditions affecting the relationship

3. **System Impact Assessment**
   - Evaluate how the new connection affects:
     * Overall system connectivity
     * System dynamics identified in Phase 3
     * Future pathways described in Phase 4
     * Market integration aspects from Phase 5

### Step 5: Connection Implementation

For each validated connection:

1. **Relationship Description**
   - Write a clear, evidence-based description of the relationship
   - Include specific citations from the documentation
   - Note any conditions or limitations

2. **Integration Check**
   - Ensure the new connection aligns with:
     * Existing system dynamics
     * Management frameworks
     * Future adaptation pathways
     * Market integration mechanisms

3. **Documentation**
   - Record the full justification for each new connection
   - Include all supporting evidence
   - Note any implications for system understanding

## Output Format

For each new connection identified:

```
Connection ID: [Unique identifier]
Isolated Variable: [Name]
Connected Variable: [Name]
Relationship Type: [Direct/Indirect]
Direction: [Positive/Negative/Variable]
Strength: [Strong/Medium/Weak]
Evidence Quality: [High/Medium/Low]
Primary Evidence: [Citations from documentation]
Relationship Description: [Clear description of the connection]
System Impact: [How this connection affects overall system understanding]
```

## Iteration Process

1. Apply this analysis to one isolated variable at a time
2. After each connection is established, reassess the system map
3. Document any emerging patterns or insights
4. Identify any remaining isolation issues
5. Repeat the process until all variables are meaningfully connected

## Quality Control

For each proposed connection, verify:
1. Evidence exists in the provided documentation
2. The relationship makes logical sense within the system
3. The connection adds meaningful understanding to the system
4. The relationship aligns with known system dynamics
5. The connection supports identified future pathways

## Documentation Requirements

Maintain detailed records of:
1. All evidence used to justify connections
2. Any assumptions made in establishing relationships
3. Uncertainties or knowledge gaps identified
4. Implications for system understanding
5. Recommendations for future research or validation
