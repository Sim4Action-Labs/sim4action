# SESSF Relationship Discovery Prompt

## Objective
To systematically identify and justify additional relationships between variables in the SESSF systems map, creating a more interconnected and comprehensive representation of the fishery system while maintaining logical consistency and evidence-based justification.

## Process Structure

### Step 1: Variable Connectivity Analysis
1. Identify isolated or weakly connected variables (fewer than 3 connections)
2. Group these variables by category (Stock, Economic, Environmental, etc.)
3. For each isolated variable:
   - List all existing connections
   - Note its primary function in the system
   - Identify potential interaction points with other variables

### Step 2: Cross-Category Connection Assessment
For each pair of categories, systematically consider:
1. Direct influences
2. Indirect influences through intermediate variables
3. Feedback loops and cyclic relationships
4. Time-delayed effects
5. Spatial interactions

### Step 3: Relationship Justification Framework
For each proposed new relationship, document:

```
From Variable: [ID and Name]
To Variable: [ID and Name]
Relationship Type: [Positive/Negative/Variable]
Strength: [Strong/Medium/Weak]
Primary Justification: [Main logical argument]
Supporting Evidence:
- Empirical: [Data-based evidence if available]
- Theoretical: [Conceptual/logical pathway]
- System Context: [How it fits in the broader system]
Confidence Level: [High/Medium/Low]
Time Scale: [Short/Medium/Long-term]
Spatial Scale: [Local/Regional/System-wide]
```

### Step 4: Relationship Validation Criteria
Each new relationship must meet these criteria:
1. **Logical Consistency**: Clear cause-and-effect pathway
2. **System Relevance**: Contributes to understanding system behavior
3. **Evidence Base**: Can be supported by data, theory, or expert knowledge
4. **Distinctiveness**: Not redundant with existing relationships
5. **Scale Appropriateness**: Operates at relevant temporal and spatial scales

### Step 5: Priority Assessment
Prioritize new relationships based on:
1. Contribution to system connectivity
2. Strength of justification
3. Strategic importance in the system
4. Data availability for validation
5. Management relevance

## Implementation Guide

### Phase 1: Initial Scan
1. Start with environmental variables connecting to stock variables
2. Move to economic-social interactions
3. Examine technical-management linkages
4. Consider indigenous knowledge connections

### Phase 2: Deep Dive
For each category pair, consider:
```
Primary Effects:
- Direct resource flows
- Information flows
- Control mechanisms
- Feedback processes

Secondary Effects:
- Indirect impacts
- Cumulative effects
- Emergent properties
- System adaptations

Cross-Scale Interactions:
- Temporal dynamics
- Spatial relationships
- Organizational levels
- Social-ecological coupling
```

### Phase 3: System Integration
1. Map new relationships onto existing network
2. Identify emerging patterns
3. Validate system-wide consistency
4. Document uncertainty and knowledge gaps

## Example Application

Example of discovering a new relationship:
```
From Variable: V43 (Chlorophyll-a)
To Variable: V31 (Primary Productivity)
Type: Positive
Strength: Strong
Primary Justification: Chlorophyll-a is a direct indicator of phytoplankton biomass and primary productivity
Supporting Evidence:
- Empirical: Satellite data showing correlation between chlorophyll-a and productivity
- Theoretical: Established biological principle in marine ecosystems
- System Context: Forms base of marine food web affecting fish stocks
Confidence: High
Time Scale: Short-term
Spatial Scale: Regional
```

## Quality Control Checklist
- [ ] Relationship is logically sound
- [ ] Evidence is clearly documented
- [ ] Scale of effect is specified
- [ ] Uncertainty is acknowledged
- [ ] System context is considered
- [ ] Management relevance is clear
- [ ] Not redundant with existing relationships
- [ ] Contributes to system understanding

## Notes
- Focus on creating meaningful connections that enhance system understanding
- Balance between comprehensiveness and parsimony
- Document uncertainty and assumptions
- Consider both direct and indirect effects
- Maintain focus on management relevance
- Consider temporal and spatial scales
- Acknowledge data limitations
