# Task 6: Context-Dependent Relationship Analysis

## Objective
Analyze and document how relationships between variables in the Southern and Eastern Scalefish and Shark Fishery (SESSF) system vary under different contexts based on the Fishery System Profile and previous tasks. Each variation should be supported by specific evidence and references, with clear documentation of its behavior and implications.

## Context
This task builds upon the relationship matrix from Task 3, feedback loops from Task 4, and indirect relationships from Task 5, focusing on understanding how relationships change under different conditions. The variations identified here will be crucial for understanding system adaptability and informing subsequent analysis tasks.

## Input
- Output from Task 3 (Relationship Matrix)
- Output from Task 4 (Feedback Loop Identification)
- Output from Task 5 (Indirect Relationship Analysis)
- Fishery System Profile for Southern and Eastern Scalefish and Shark Fishery (SESSF)
- Results from previous iterations (if applicable)

## Process

1. Context Identification:
   - Analyze profile for contexts
   - Identify context types
   - Note context boundaries
   - Document context factors
   - Include evidence base

2. Evidence Collection:
   - For each context:
     - Extract relevant evidence from profile
     - Note specific examples or case studies
     - Document supporting references
     - Identify temporal patterns
     - Note spatial/contextual factors

3. Variation Analysis:
   - For each context:
     - Document relationship changes
     - Note strength variations
     - Include direction changes
     - Add evidence citations
     - Note temporal aspects
     - Document spatial factors

4. Behavior Analysis:
   - For each variation:
     - Analyze variation behavior
     - Note stability characteristics
     - Document oscillation patterns
     - Include threshold effects
     - Note non-linear behaviors

5. Impact Assessment:
   - For each variation:
     - Evaluate ecological impacts
     - Assess economic implications
     - Consider social effects
     - Note management challenges
     - Document adaptation strategies

6. Variation Documentation:
   - For each variation:
     - Create detailed descriptions
     - List affected variables
     - Document evidence
     - Note exceptions and conditions
     - Include impact assessments

7. Quality Control:
   - Verify variation completeness
   - Check behavior accuracy
   - Validate evidence citations
   - Ensure logical coherence
   - Confirm consistency with profile
